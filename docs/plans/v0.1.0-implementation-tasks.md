## 0. Read-Before-You-Code Conventions

These rules apply to every task. **Do not skip.**

1. **Ruby version:** `>= 4.0`. Use `Data.define`, pattern matching, endless methods (`def foo = ...`) where they improve clarity.
2. **Keyword arguments everywhere in the public API.** No positional public method args. Internal helpers may use positional.
3. **Frozen literals:** add `# frozen_string_literal: true` to every `.rb` file.
4. **No `require_relative` chains in user-facing code.** Use top-level `require` in `lib/brightdata.rb`. All other files use `require "brightdata/..."`.
5. **YARD on every public method, class, module, and parameter.** Use `@param`, `@return`, `@raise`, `@example`, `@note`, `@api private`.
6. **Minitest assertions:** `assert_equal expected, actual` (expected first). Use `assert_raises(Klass) { ... }` for error checks. One assertion concept per test method.
7. **WebMock:** `WebMock.disable_net_connect!(allow_localhost: true)` in `test_helper.rb`. All HTTP stubbed.
8. **Errors:** use the gem hierarchy from `BrightData::Errors` (Task 7). Never raise stdlib `::ArgumentError` from user-facing code — raise `BrightData::ArgumentError`.
9. **`raw:` field contract:** every output `Data.define` class has a `:raw` field that holds the entire parsed-hash response object. Document in YARD with `@note Use #raw to access fields not yet typed by this gem.`
10. **Commits:** one commit per task. Conventional Commits format (`feat:`, `chore:`, `test:`, `docs:`, `refactor:`). Reference the task number in the body: `Task 12: ...`.
11. **CI must stay green** after every task. Lint + tests + YARD.
12. **No new runtime gems** beyond what's in `tech-stack.md` (`simple-result` only). Adding any other runtime dep requires updating `tech-stack.md` first.

```ruby
# lib/brightdata/errors.rb
# frozen_string_literal: true

module BrightData
  # Base error class. Rescue this to catch any BrightData failure.
  class Error < StandardError; end

  # Raised when the gem is misconfigured (e.g., missing api_token).
  class ConfigurationError < Error; end

  # Raised when a caller passes an invalid argument shape or value.
  # Inherits from {BrightData::Error}, NOT from ::ArgumentError, so
  # `rescue ::ArgumentError` will NOT catch this. Use `rescue BrightData::ArgumentError`
  # or `rescue BrightData::Error`.
  class ArgumentError < Error; end

  # Raised on 401/403 responses from Bright Data API.
  class AuthError < Error; end

  # Base class for transport-level errors. Carries status, body, and raw response.
  class HTTPError < Error
    # @return [Integer, nil] HTTP status (nil if request never completed)
    attr_reader :status
    # @return [String, nil] raw response body
    attr_reader :body
    # @return [Net::HTTPResponse, nil] raw Net::HTTP response
    attr_reader :response

    # @param message [String]
    # @param status [Integer, nil]
    # @param body [String, nil]
    # @param response [Net::HTTPResponse, nil]
    def initialize(message, status: nil, body: nil, response: nil)
      super(message)
      @status = status
      @body = body
      @response = response
    end
  end

  # Raised on 429 Too Many Requests.
  class RateLimitError < HTTPError
    # @return [Integer, nil] value of the `Retry-After` header in seconds (nil if absent)
    attr_reader :retry_after

    def initialize(message, status: 429, body: nil, response: nil, retry_after: nil)
      super(message, status:, body:, response:)
      @retry_after = retry_after
    end
  end

  # Raised on 5xx responses.
  class ServerError < HTTPError; end

  # Raised when /scrape exceeds Bright Data's 60s synchronous cap.
  # The {snapshot_id} is preserved so the caller can switch to
  # `Snapshot.new(id: e.snapshot_id, http: client.http).wait`.
  class ScrapeTimeoutError < Error
    # @return [String] snapshot_id returned by Bright Data
    attr_reader :snapshot_id
    def initialize(message, snapshot_id:)
      super(message)
      @snapshot_id = snapshot_id
    end
  end

  # Raised by Snapshot#wait when the snapshot terminates in :failed status.
  # NOTE: Snapshot#wait normally returns SimpleResult::Failure on :failed —
  # this error class exists for callers who explicitly want exception semantics
  # via `Snapshot#wait!` (deferred to v0.2).
  class SnapshotFailedError < Error
    attr_reader :snapshot_id, :details
    def initialize(message, snapshot_id:, details: nil)
      super(message)
      @snapshot_id = snapshot_id
      @details = details
    end
  end
end
```

**Tests:** instantiate each error class, assert attr_readers, assert inheritance chain.

```ruby
# lib/brightdata/http.rb
# frozen_string_literal: true

require "net/http"
require "uri"
require "json"

module BrightData
  # Thin Net::HTTP wrapper. Single point of egress for the gem.
  # Encapsulated so a future v0.2+ migration to Faraday is a one-file change.
  class HTTP
    BASE_URL = "https://api.brightdata.com"

    # Must be > Bright Data's /scrape 60s API cap so the fallback
    # {snapshot_id} body is received instead of a TCP timeout swallowing it.
    DEFAULT_TIMEOUT = 90

    # @param api_token [String] Bright Data API token
    # @param base_url [String] override for testing
    # @param open_timeout [Integer] TCP open timeout in seconds
    # @param read_timeout [Integer] socket read timeout in seconds
    # @param logger [Logger, nil] optional logger for :debug request traces
    # @raise [BrightData::ConfigurationError] if api_token is nil/empty
    def initialize(api_token:, base_url: BASE_URL, open_timeout: 10, read_timeout: DEFAULT_TIMEOUT, logger: nil)
      raise ConfigurationError, "api_token is required" if api_token.nil? || api_token.empty?
      @api_token = api_token
      @base_url = base_url
      @open_timeout = open_timeout
      @read_timeout = read_timeout
      @logger = logger
    end

    # POST a JSON body.
    # @param path [String] e.g. "/datasets/v3/trigger"
    # @param query [Hash] query string params
    # @param body [Hash, nil] JSON body (will be encoded)
    # @return [Hash, Array] parsed JSON
    # @raise [BrightData::AuthError, BrightData::RateLimitError, BrightData::ServerError, BrightData::HTTPError]
    def post(path:, query: {}, body: nil)
      request(method: :post, path:, query:, body:)
    end

    # GET a path.
    # @param path [String]
    # @param query [Hash]
    # @return [Hash, Array] parsed JSON
    def get(path:, query: {})
      request(method: :get, path:, query:)
    end

    private

    def request(method:, path:, query: {}, body: nil)
      uri = build_uri(path:, query:)
      req = build_request(method:, uri:, body:)
      started = Process.clock_gettime(Process::CLOCK_MONOTONIC)
      response = perform(uri:, req:)
      log_request(method:, path:, status: response.code.to_i, duration: Process.clock_gettime(Process::CLOCK_MONOTONIC) - started)
      handle_response(response)
    end

    def build_uri(path:, query:)
      uri = URI.parse("#{@base_url}#{path}")
      uri.query = URI.encode_www_form(query) unless query.empty?
      uri
    end

    def build_request(method:, uri:, body:)
      klass = (method == :post) ? Net::HTTP::Post : Net::HTTP::Get
      req = klass.new(uri)
      req["Authorization"] = "Bearer #{@api_token}"
      req["Content-Type"] = "application/json"
      req["Accept"] = "application/json"
      req.body = JSON.generate(body) if body
      req
    end

    def perform(uri:, req:)
      Net::HTTP.start(uri.host, uri.port, use_ssl: true, open_timeout: @open_timeout, read_timeout: @read_timeout) do |http|
        http.request(req)
      end
    rescue Net::OpenTimeout, Net::ReadTimeout => e
      raise HTTPError.new("Timeout: #{e.message}", status: nil, body: nil)
    end

    def handle_response(response)
      status = response.code.to_i
      body_text = response.body.to_s

      case status
      when 200..299
        parse_json(body_text)
      when 401, 403
        raise AuthError, "Bright Data API rejected the token (status #{status}): #{body_text}"
      when 429
        raise RateLimitError.new(
          "Bright Data rate limit hit (status 429)",
          status:,
          body: body_text,
          response:,
          retry_after: parse_retry_after(response)
        )
      when 500..599
        raise ServerError.new("Bright Data server error (status #{status})", status:, body: body_text, response:)
      else
        raise HTTPError.new("Unexpected HTTP status #{status}", status:, body: body_text, response:)
      end
    end

    def parse_json(text)
      return nil if text.empty?
      JSON.parse(text, symbolize_names: true)
    end

    def parse_retry_after(response)
      raw = response["retry-after"] or response["Retry-After"]
      Integer(raw, 10) if raw && raw.match?(/\A\d+\z/)
    end

    def log_request(method:, path:, status:, duration:)
      return unless @logger
      @logger.debug { format("[brightdata] %s %s -> %d in %.3fs", method.upcase, path, status, duration) }
    end
  end
end
```

**Test scenarios (≥ 12):**
- happy path 200 returns parsed hash
- 200 with empty body returns nil
- 200 with array body returns array
- query params encoded in URL
- Authorization header is `Bearer <token>` exactly
- 401 → AuthError
- 403 → AuthError
- 429 → RateLimitError with `retry_after` (test with header `Retry-After: 30`)
- 429 → RateLimitError with `retry_after: nil` (header absent)
- 500 → ServerError
- 502 → ServerError
- Net::ReadTimeout → HTTPError(status: nil)

```ruby
# lib/brightdata/snapshot.rb
# frozen_string_literal: true

module BrightData
  # Represents a Bright Data Dataset job (a snapshot). Returned by every
  # `.trigger` call. Use {#wait} to block until the snapshot reaches a
  # terminal status (:ready or :failed).
  #
  # @example Wait for results
  #   snapshot = client.linkedin.profiles.trigger(urls: ["https://..."])
  #   result = snapshot.wait
  #   case result
  #   in success: { value: }
  #     value # => Array of BrightData::LinkedIn::Types::Profile
  #   in failure: { value: }
  #     value # => Hash with error details
  #   end
  class Snapshot
    STATUSES = %i[starting running ready failed].freeze
    TRIGGER_RESPONSE_KEY = :snapshot_id

    PROGRESS_PATH_TEMPLATE = "/datasets/v3/progress/%s"
    RESULTS_PATH_TEMPLATE = "/datasets/v3/snapshot/%s"
    DEFAULT_TIMEOUT = 300
    DEFAULT_POLL_INTERVAL = 5

    # @return [String] snapshot ID
    attr_reader :id
    # @return [Symbol, nil] latest known status (:starting/:running/:ready/:failed). nil until first progress call.
    attr_reader :status

    # @param id [String] snapshot_id from /datasets/v3/trigger
    # @param http [BrightData::HTTP]
    # @param result_parser [#call] callable raw_array -> parsed
    def initialize(id:, http:, result_parser: ->(raw) { raw })
      @id = id
      @http = http
      @result_parser = result_parser
      @status = :starting
    end

    # Poll progress once. Updates {#status} and returns the new value.
    # @return [Symbol]
    def progress
      payload = @http.get(path: format(PROGRESS_PATH_TEMPLATE, @id))
      raw = payload.fetch(:status)
      sym = raw.to_sym
      unless STATUSES.include?(sym)
        raise Error, "Unknown snapshot status from API: #{raw.inspect}"
      end
      @status = sym
    end

    # @return [Boolean]
    def ready? = @status == :ready
    # @return [Boolean]
    def failed? = @status == :failed

    # Block until snapshot reaches a terminal status or timeout elapses.
    #
    # @param timeout [Numeric] seconds to wait before raising. Pass `Float::INFINITY` for unbounded.
    # @param poll_interval [Numeric] seconds between progress polls
    # @return [BrightData::Result::Success<Array>] when status reaches :ready, value is parsed results
    # @return [BrightData::Result::Failure<Hash>] when status reaches :failed, value is the progress payload
    # @raise [BrightData::ScrapeTimeoutError] when timeout elapses while still :starting/:running
    def wait(timeout: DEFAULT_TIMEOUT, poll_interval: DEFAULT_POLL_INTERVAL)
      deadline = monotonic_now + timeout
      loop do
        progress
        return Result.success(@result_parser.call(results)) if ready?
        return Result.failure(failure_payload) if failed?
        if monotonic_now >= deadline
          raise ScrapeTimeoutError.new(
            "Snapshot #{@id} still :#{@status} after #{timeout}s",
            snapshot_id: @id
          )
        end
        sleep_for(poll_interval)
      end
    end

    # Fetch raw results array. Caller usually uses {#wait} instead.
    # @return [Array]
    def results
      @http.get(path: format(RESULTS_PATH_TEMPLATE, @id), query: { format: "json" })
    end

    private

    def failure_payload
      @http.get(path: format(PROGRESS_PATH_TEMPLATE, @id))
    end

    # Extracted for test stubability. See finding E3 in gauntlet review.
    def sleep_for(seconds)
      sleep(seconds)
    end

    def monotonic_now
      Process.clock_gettime(Process::CLOCK_MONOTONIC)
    end
  end
end
```

```ruby
# lib/brightdata/linkedin/endpoint.rb
# frozen_string_literal: true

module BrightData
  module LinkedIn
    # Mixin providing shared trigger/scrape helpers for LinkedIn endpoint classes.
    # Including class must:
    #   - assign @http in its initializer
    #   - override #result_parser to return a Proc that maps raw array -> typed objects
    # @api private
    module Endpoint
      TRIGGER_PATH = "/datasets/v3/trigger"
      SCRAPE_PATH  = "/datasets/v3/scrape"

      private

      # @param dataset_key [Symbol] key in BrightData::Datasets::LINKEDIN
      # @param inputs [Array<Data>] input Data instances
      # @param extra_query [Hash] additional query params (e.g., {type: "discover_new"})
      # @return [BrightData::Snapshot]
      def trigger_with(dataset_key:, inputs:, extra_query: {})
        payload = @http.post(
          path: TRIGGER_PATH,
          query: query_for(dataset_key:, extra: extra_query),
          body: { input: serialize_inputs(inputs) }
        )
        Snapshot.new(
          id: payload.fetch(:snapshot_id),
          http: @http,
          result_parser: result_parser
        )
      end

      # @return [Array] parsed results (typed by result_parser)
      # @raise [BrightData::ScrapeTimeoutError] when API returns snapshot_id fallback
      def scrape_with(dataset_key:, inputs:, extra_query: {})
        payload = @http.post(
          path: SCRAPE_PATH,
          query: query_for(dataset_key:, extra: extra_query),
          body: { input: serialize_inputs(inputs) }
        )
        if payload.is_a?(Hash) && payload[:snapshot_id]
          raise ScrapeTimeoutError.new(
            "Bright Data /scrape exceeded its 60s cap. Use .trigger + Snapshot#wait, " \
            "or recover via Snapshot.new(id: e.snapshot_id, http: client.http).wait",
            snapshot_id: payload[:snapshot_id]
          )
        end
        result_parser.call(payload)
      end

      # Override in including class to map raw API hashes to typed Data instances.
      # @return [#call]
      def result_parser
        ->(raw) { raw }
      end

      def query_for(dataset_key:, extra:)
        { dataset_id: Datasets.id_for(dataset_key) }.merge(extra)
      end

      def serialize_inputs(inputs)
        inputs.map do |input|
          input.respond_to?(:to_api_hash) ? input.to_api_hash : input.to_h
        end
      end
    end
  end
end
```
